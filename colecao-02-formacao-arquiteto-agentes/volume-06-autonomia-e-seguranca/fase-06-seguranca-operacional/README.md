# Fase 06 — Segurança Operacional

## Objetivo

Estudar mecanismos de proteção para sistemas orientados por agentes e prevenir falhas operacionais.

## Assuntos principais

- Guards
- Escopo
- Permissões
- RBAC
- ACL
- Prompt injection
- Data leakage
- Tenant isolation

## Como usar este diretório

- `prompt.md`: prompt-base para estudar segurança operacional.
- `exercicios.md`: espaço para exercícios de ameaça, controle e mitigação.
- `notas.md`: espaço para registrar riscos, controles e exemplos.

## Entregáveis esperados

- Mapa de ameaças e controles.
- Definição de escopos, permissões e isolamento.
- Síntese sobre prevenção de comportamentos indesejados.

## Critérios de avanço

- Explicar controles de permissão e isolamento.
- Identificar riscos específicos de agentes e IA.
- Propor guards e limites compatíveis com o contexto operacional.

## Observações

Segurança operacional deve acompanhar autonomia. Controles precisam ser claros, testáveis e auditáveis.
