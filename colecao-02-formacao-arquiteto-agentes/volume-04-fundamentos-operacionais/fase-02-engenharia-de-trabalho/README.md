# Fase 02 — Engenharia de Trabalho

## Objetivo

Aprender a transformar problemas, ideias e objetivos em trabalho executável, rastreável e priorizado.

## Assuntos principais

- Requisitos
- Specs
- Tasks
- Backlog
- Priorização
- Reports
- Execution logs
- Rastreabilidade

## Como usar este diretório

- `prompt.md`: prompt-base para estudar engenharia de trabalho.
- `exercicios.md`: espaço para práticas de decomposição e planejamento.
- `notas.md`: espaço para registrar decisões, dúvidas e modelos.

## Entregáveis esperados

- Exemplo de transformação de problema em especificação.
- Backlog inicial com tarefas rastreáveis.
- Relatório simples de execução e critérios de acompanhamento.

## Critérios de avanço

- Diferenciar requisito, especificação e tarefa.
- Priorizar trabalho com justificativa clara.
- Manter rastreabilidade entre objetivo, execução e relatório.

## Observações

O foco é tornar trabalho claro o suficiente para ser executado, validado e melhorado.
