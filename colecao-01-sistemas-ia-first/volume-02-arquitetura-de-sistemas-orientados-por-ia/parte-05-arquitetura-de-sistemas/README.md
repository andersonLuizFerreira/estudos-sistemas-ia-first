# Parte 05 — Arquitetura de Sistemas

## Objetivo

Aprender a projetar sistemas complexos por meio da análise de estilos arquiteturais e seus trade-offs.

## Assuntos principais

- Monólitos
- Modular monolith
- SOA
- Microserviços
- Event-driven architecture
- APIs
- Mensageria
- Cache
- Escalabilidade

## Como usar este diretório

- `prompt.md`: prompt-base para conduzir o estudo de arquitetura.
- `exercicios.md`: espaço para exercícios de comparação e desenho arquitetural.
- `notas.md`: espaço para registrar decisões, trade-offs e exemplos.

## Entregáveis esperados

- Comparativo entre estilos arquiteturais.
- Registro de trade-offs de escalabilidade, manutenção e integração.
- Esboços de arquitetura para cenários reais.

## Critérios de avanço

- Explicar vantagens e limitações de cada estilo.
- Justificar escolhas arquiteturais com base no problema.
- Relacionar arquitetura tradicional com sistemas IA-first.

## Observações

Evite respostas absolutas. Arquitetura exige análise de contexto, restrições e consequências.
