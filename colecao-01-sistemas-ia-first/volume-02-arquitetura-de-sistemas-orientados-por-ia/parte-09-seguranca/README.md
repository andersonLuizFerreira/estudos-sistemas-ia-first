# Parte 09 — Segurança

## Objetivo

Aprender fundamentos de segurança aplicados a sistemas modernos e sistemas IA-first.

## Assuntos principais

- Segurança tradicional
- Autenticação
- Autorização
- RBAC
- ACL
- Tenant isolation
- Prompt injection
- Data leakage
- RAG poisoning

## Como usar este diretório

- `prompt.md`: prompt-base para conduzir o estudo de segurança.
- `exercicios.md`: espaço para exercícios de análise de risco e proteção.
- `notas.md`: espaço para registrar ameaças, controles e dúvidas.

## Entregáveis esperados

- Mapa de ameaças para sistemas IA-first.
- Comparação entre controles tradicionais e riscos específicos de IA.
- Síntese de estratégias para reduzir danos operacionais.

## Critérios de avanço

- Explicar autenticação, autorização e isolamento.
- Identificar riscos como prompt injection, vazamento de dados e poisoning.
- Propor controles compatíveis com sistemas orientados por IA.

## Observações

Segurança deve ser estudada como responsabilidade arquitetural contínua, não como etapa final.
