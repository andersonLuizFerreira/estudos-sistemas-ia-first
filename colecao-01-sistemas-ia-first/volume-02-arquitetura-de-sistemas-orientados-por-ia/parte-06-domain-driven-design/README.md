# Parte 06 — Domain Driven Design

## Objetivo

Aprender modelagem de domínio como base para projetar sistemas complexos antes da implementação.

## Assuntos principais

- Domínio
- Subdomínio
- Bounded context
- Linguagem ubíqua
- Eventos de domínio
- Agregados
- Entidades
- Value objects

## Como usar este diretório

- `prompt.md`: prompt-base para estudar modelagem de domínio.
- `exercicios.md`: espaço para exercícios de modelagem e análise de casos.
- `notas.md`: espaço para registrar linguagem, decisões e exemplos.

## Entregáveis esperados

- Mapa de domínio para exemplos como ERP, marketplace, SaaS ou indústria.
- Identificação de bounded contexts.
- Registro de eventos, entidades, agregados e value objects.

## Critérios de avanço

- Diferenciar domínio, subdomínio e bounded context.
- Construir uma linguagem ubíqua coerente.
- Explicar como a modelagem orienta arquitetura e implementação.

## Observações

Mantenha foco no negócio e no domínio. Código é consequência da modelagem, não ponto de partida.
