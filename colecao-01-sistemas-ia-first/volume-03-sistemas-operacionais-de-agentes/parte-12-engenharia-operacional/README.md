# Parte 12 — Engenharia Operacional

## Objetivo

Estudar como transformar agentes em um sistema operacional de trabalho com processos, artefatos e governança.

## Assuntos principais

- Specs
- Tasks
- Workflows
- Skills
- Reports
- Runtime
- Governança

## Como usar este diretório

- `prompt.md`: prompt-base para estudar operacionalização de agentes.
- `exercicios.md`: espaço para exercícios de planejamento e execução.
- `notas.md`: espaço para registrar artefatos, fluxos e aprendizados.

## Entregáveis esperados

- Modelo de fluxo entre especificação, tarefa, execução e relatório.
- Registro de artefatos operacionais essenciais.
- Síntese sobre governança aplicada ao trabalho de agentes.

## Critérios de avanço

- Explicar como trabalho é transformado em execução rastreável.
- Relacionar workflows, skills e runtime.
- Definir critérios mínimos de controle operacional.

## Observações

O foco é operação real. Agentes precisam de processo, rastreabilidade e critérios de qualidade.
