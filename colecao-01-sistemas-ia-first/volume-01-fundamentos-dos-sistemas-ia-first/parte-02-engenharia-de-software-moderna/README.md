# Parte 02 — Engenharia de Software Moderna

## Objetivo

Construir uma base de engenharia de software para compreender como sistemas profissionais são concebidos, mantidos e evoluídos.

## Assuntos principais

- Requisitos
- Casos de uso
- Modelagem
- Arquitetura
- Testes
- Governança
- Qualidade
- Manutenibilidade
- Evolução de sistemas

## Como usar este diretório

- `prompt.md`: prompt-base para conduzir o estudo desta parte.
- `exercicios.md`: espaço para exercícios, estudos de caso e práticas.
- `notas.md`: espaço para registrar conceitos, dúvidas e decisões de estudo.

## Entregáveis esperados

- Síntese dos fundamentos de engenharia de software.
- Comparação entre teoria, prática, mercado e academia.
- Registro de erros comuns e boas práticas.

## Critérios de avanço

- Diferenciar requisitos, modelagem, arquitetura e testes.
- Explicar como qualidade e governança afetam sistemas reais.
- Relacionar engenharia de software com sistemas orientados por agentes.

## Observações

Evite tratar IA como substituta da engenharia. Esta parte consolida disciplina técnica antes de avançar.
